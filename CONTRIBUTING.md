# Contributing to Concord

## Getting Started

See DEVELOPMENT.md for local setup.

## Submitting Changes

1. Fork the repo
2. Create a feature branch (see DEVELOPMENT.md for naming)
3. Write tests for your changes
4. Open a PR against develop
5. Other team member reviews; Claude Code as co-reviewer

## Standards

Python 3.11+, type hints on all public functions.
ruff for linting — no warnings allowed on merge.
Commit format: feat: fix: docs: chore: test:
